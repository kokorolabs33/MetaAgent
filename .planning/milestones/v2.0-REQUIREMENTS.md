# Requirements: TaskHub

**Defined:** 2026-04-04
**Core Value:** Developers can experience a complete A2A multi-agent collaboration flow in a polished, open-source package

## v1 Requirements (Milestone v1.0 — Complete)

### Foundation (GitHub Readiness)

- [x] **FOUND-01**: Frontend-backend interaction bugs are fixed and all core flows work without errors
- [x] **FOUND-02**: SSE subscribe/replay race condition is fixed — no dropped events or stuck DAG nodes
- [x] **FOUND-03**: One-click local startup via `docker compose up` with seeded demo agents and pre-configured database
- [x] **FOUND-04**: Comprehensive README with hero screenshot/GIF, badges, architecture diagram, and quick-start instructions

### Observability

- [x] **OBSV-01**: Agent status indicators show online/working/idle/offline states with color-coded dots and pulse animation when working
- [x] **OBSV-02**: Agent status changes are pushed via SSE on a global agent status channel (not polled)
- [x] **OBSV-03**: Subtask timeline/trace view shows chronological execution with agent name, duration, and output per subtask
- [x] **OBSV-04**: Adaptive replanning events are visible in the UI with "replanned because X" notifications and old vs new subtask comparison

### Interaction

- [x] **INTR-01**: User can send messages to specific sub-agents during active task execution via @mention in chat (advisory mode)
- [x] **INTR-02**: Multi-task parallel view dashboard shows multiple running tasks simultaneously with status badges and active agent indicators
- [x] **INTR-03**: Multi-task SSE connection strategy handles browser connection limits (multiplexed endpoint)
- [x] **INTR-04**: Dashboard supports task filtering by status (pending/running/completed/failed) and search by title

### Templates

- [x] **TMPL-01**: User can save a successful task's orchestration pattern as a reusable template with variable parameterization
- [x] **TMPL-02**: New tasks automatically match against existing templates and suggest relevant ones based on similarity
- [x] **TMPL-03**: Template execution statistics are recorded and displayed (usage count, success rate, average duration)

### Demo Readiness

- [x] **DEMO-01**: Task decomposition works with OpenAI API key (not just Anthropic)
- [x] **DEMO-02**: /manage/templates shows seeded template data with usage stats
- [x] **DEMO-03**: /manage/analytics has per-agent drill-down with task assignments and time/status filters
- [x] **DEMO-04**: /manage/audit shows audit logs with filtering by time, agent, and event type
- [x] **DEMO-05**: /manage/settings/policies shows seeded policy data
- [x] **DEMO-06**: Demo data makes pages look populated and functional

## v2 Requirements (Milestone v2.0 — Wow Moment)

Requirements for transforming agents from chat-only to real-work-capable. Each maps to roadmap phases.

### Agent Tool Use

- [ ] **TOOL-01**: Agent supports OpenAI function calling loop — can invoke tools and process tool_calls responses
- [ ] **TOOL-02**: Agent has built-in web search tool (Tavily API) to search real-time data
- [ ] **TOOL-03**: Tool call events pushed via SSE to frontend — UI shows which tool agent is calling
- [ ] **TOOL-04**: Different agent roles have different available tool sets (e.g., Engineering has code analysis, Marketing has search)

### Artifact Rendering

- [ ] **ARTF-01**: Typed artifact schema (search_results, code, table, data) with dedicated renderers per type
- [ ] **ARTF-02**: Frontend rich card components render search results, code blocks, tables, data cards
- [ ] **ARTF-03**: Artifacts support copy to clipboard and download as file
- [ ] **ARTF-04**: Markdown rendering upgraded to remark-gfm + rehype-highlight with GFM tables and code highlighting

### Streaming Output

- [ ] **STRM-01**: Agent replies stream in real-time (token-level) via SSE delta events to frontend
- [ ] **STRM-02**: Streaming output renders markdown progressively (tables, code blocks form as tokens arrive)

### Inbound Webhooks

- [ ] **HOOK-01**: HMAC-SHA256 authenticated webhook endpoint — external events trigger task creation
- [ ] **HOOK-02**: GitHub and Slack webhook payload parsers automatically convert to TaskHub tasks
- [ ] **HOOK-03**: Frontend webhook management page (create/edit/delete/view webhook configurations)
- [ ] **HOOK-04**: Idempotency protection — duplicate requests do not create duplicate tasks

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| OpenTelemetry / external observability integration | LangFuse/LangSmith already do this; weeks of work for zero differentiation |
| Visual workflow builder (drag-and-drop DAG editor) | n8n has 150k+ stars on this; TaskHub's value is LLM-driven DAG creation |
| Multi-tenancy / org management | SaaS-tier feature; deferred to enterprise milestone |
| Plugin/extension system for custom agent types | Premature generalization; A2A protocol itself is the extension mechanism |
| Evaluation / LLM-as-judge scoring | Langfuse has full eval infrastructure; no demo payoff for TaskHub |
| Mobile-responsive UI | Desktop-first developer tool; don't break mobile but don't optimize |
| Agent marketplace / registry | Premature; needs community adoption first; agent-card.json is the discovery mechanism |
| Real-time multi-user collaboration | Complex WebSocket infrastructure for a feature that barely matters in single-developer showcase |
| A2UI rendering | Nascent Google-specific standard; typed artifact parts are lighter and fully controlled |
