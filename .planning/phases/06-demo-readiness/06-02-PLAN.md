      - "Market Analysis: AI Developer Tools 2026" (completed, 240s duration)

      - "Bug Triage: Session Timeout Issue" (completed, 90s duration)

      - "Compliance Review: GDPR Data Processing" (failed, 0s duration)
      - "Infrastructure Cost Optimization" (completed, 200s duration)
      - "API Documentation Update" (completed, 150s duration)

   Each task: deterministic UUID (e.g., `"demo-task-001"` through `"demo-task-008"`), `created_by = "local-user"`, `source = "web"`.
   Set `created_at` and `completed_at` using `NOW() - INTERVAL 'Nd'` with varying day offsets (1-14 days).
   The failed task has `completed_at = NULL` and `error = 'Agent unreachable after 3 retries'`.

   b) **2-4 subtasks per task** (16-24 total) assigned to agent names resolved via subquery:
      ```sql
      INSERT INTO subtasks (id, task_id, instruction, agent_id, status, output, depends_on, ...)
      VALUES ($1, $2, $3, (SELECT id FROM agents WHERE name = $4 LIMIT 1), $5, $6, $7, ...)
      ON CONFLICT (id) DO NOTHING
      ```
      Use agent names like 'Engineering Agent', 'Finance Agent', 'Legal Agent', 'Marketing Agent' (these are the default agents from the openaiagent roles).
      If the agent name subquery returns NULL (agent not registered), use a fallback empty string.
      Subtask statuses: most 'completed', some 'failed' (for the failed task).

   c) **Events for each task** (30-50 total): `task.planning`, `task.planned`, `task.running`, `subtask.created`, `agent.working`, `message`, `task.completed` (or `task.failed`).
      Each event: deterministic UUID, `actor_type = 'system'` or `'agent'`, `data = '{}'::jsonb`.
      Timestamps offset relative to task's `created_at`.

   d) **Template executions** linking 4 of the completed tasks to seeded templates:
      - Task 001 -> "Security Audit Checklist" template, outcome "completed", duration 180s
      - Task 002 -> "Market Research Report" template, outcome "completed", duration 240s
      - Task 003 -> "Code Review Pipeline" template, outcome "completed", duration 120s
      - Task 005 -> "Content Creation Pipeline" template, outcome "completed", duration 300s

      Template IDs reference the deterministic IDs from templates.go (e.g., `"tmpl-security-audit"`).
      Template execution INSERT: `ON CONFLICT (id) DO NOTHING`.

   Use a single transaction (`pool.Begin`) for all inserts. If any error occurs, rollback. Log progress: `log.Printf("seed-demo: seeded %d tasks, %d subtasks, %d events, %d template executions", ...)`.
