<objective>
Add streaming OpenAI support to the agent binary and create the platform-side relay endpoint so token deltas flow from the LLM through to the SSE broker in real time.

Purpose: This is the backend pipeline for STRM-01 (real-time token-level streaming). The agent calls OpenAI with `stream: true`, batches tokens, and POSTs each batch to the platform server. The platform server publishes these as ephemeral `agent.streaming_delta` SSE events via the broker (not persisted to DB). The final complete message is still persisted as a normal `message` event by the executor after the agent finishes.

Output: Streaming delta pipeline from OpenAI API through agent binary to platform SSE broker. New files: `cmd/openaiagent/stream.go` (streaming client), `internal/handlers/streaming_delta.go` (callback endpoint).
</objective>

      - Adds `"stream": true` and `"stream_options": {"include_usage": true}` to the `toolChatRequest` JSON
      - Reads the response body line-by-line using `bufio.Scanner` (SSE format: lines starting with `data: `)

This reduces event volume by ~10-20x while still feeling real-time.

Only call `StreamDeltaHook` during text generation chunks, not during tool execution.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent -> Platform (internal callback) | Agent binary POSTs deltas to platform server; currently no auth on internal endpoint |
| Platform SSE -> Browser | Ephemeral delta events flow to browser via existing SSE channel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | POST /api/internal/streaming-delta | accept | Internal endpoint on localhost; agents run on same host. If deployed multi-host, add shared secret header validation. Acceptable for MVP/demo. |
| T-09-02 | Tampering | delta_text payload | accept | Delta text is displayed as markdown via react-markdown (safe by default, no raw HTML). Attacker-controlled delta text cannot execute XSS. |
| T-09-03 | Denial of Service | Broker channel overflow | mitigate | Token batching (~50ms / 20 chars) reduces event volume 10-20x vs per-token. Broker's existing drop-on-full prevents memory exhaustion. |
| T-09-04 | Information Disclosure | conversation_id in delta events | accept | Same trust boundary as existing SSE events; conversation_id already exposed in all event types. |
| T-09-05 | Elevation of Privilege | Internal endpoint accessible externally | mitigate | Route is /api/internal/* prefix; document that reverse proxy should block /api/internal/* from external access. |
</threat_model>
