## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-normal) |
| Label | 12px (text-xs) | 500 (medium) | 1.5 (leading-normal) |

## Color

All colors reference shadcn CSS custom properties. The app runs in dark mode only (html class="dark").

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / oklch(0.145 0 0) | Page background (`bg-background` or `bg-gray-950`) |
| Secondary (30%) | `bg-gray-900/50` + `border-border` | Task cards, tab bar container, summary row |
| Accent (10%) | `var(--primary)` / oklch(0.922 0 0) | Active tab underline, "New Task" button fill |
| Destructive | `var(--destructive)` / oklch(0.704 0.191 22.216) | Failed status badge only |

Accent reserved for:
- Active filter tab indicator (underline or background highlight)
- "New Task" primary CTA button
- Focus rings on interactive elements (search input, tab buttons)

### Grid Breakpoints (Claude's Discretion)

| Breakpoint | Columns | Gap | Reasoning |
|------------|---------|-----|-----------|
| < 640px (sm) | 1 | 16px (gap-4) | Single column on narrow content area |
| 640px - 1023px (md) | 2 | 16px (gap-4) | Two columns for medium widths |
| >= 1024px (lg) | 3 | 16px (gap-4) | Three columns for comfortable card sizing |

Tailwind classes: `grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3`

Note: Content area is already constrained to `flex-1` (viewport minus 256px sidebar), so breakpoints refer to the main content width, not full viewport.

- 12 tasks per page (fills 4 rows of 3 columns)
- Pagination appears below the grid when `totalPages > 1`
- URL query param: `?page=N` combined with `?status=X&q=Y`

## Interaction Contract

### Tab Filtering

| Action | Result |
|--------|--------|
| Click "All" tab | Load all tasks, update URL to `/?status=` |
| Click "Running" tab | Load running tasks only, update URL to `/?status=running` |
| Click "Completed" tab | Load completed tasks, update URL to `/?status=completed` |
| Click "Failed" tab | Load failed tasks, update URL to `/?status=failed` |
| Type in search box | Debounce 300ms, then load tasks with `q=` param, update URL to `/?q=search_term` |
| Change page | Load page N, update URL to `/?page=N` |

URL params are combinable: `/?status=running&q=deploy&page=2`

### Real-Time Updates (SSE)

| SSE Source | Purpose | Store |
|------------|---------|-------|
| `/api/tasks/stream?ids=a,b,c` | Multiplexed task events for visible cards | New dashboard slice in TaskStore or dedicated useDashboardStore |
| `/api/agents/stream` | Agent status dots on cards | Existing useAgentStore.agentStatuses |

- On mount: Fetch task list via API, then connect multiplexed SSE for visible task IDs
- On filter change: Disconnect SSE, fetch new list, reconnect SSE with new IDs
- On unmount: Disconnect both SSE connections

### Card Click

| Action | Result |
|--------|--------|
| Click anywhere on card | `router.push(\`/tasks/\${task.id}\`)` — navigates to full task detail |
| Card hover | `hover:bg-card/80` transition (matches existing TaskCard pattern) |

### Loading States

| State | Visual |
|-------|--------|
| Initial load | `Loader2` spinner centered in content area (matches agent health page pattern) |
| Tab/filter change | Keep existing cards visible, show subtle top-bar loading indicator (opacity pulse on filter bar) |
| SSE reconnecting | No visual change; EventSource auto-reconnects |

### Error States

| State | Visual |
|-------|--------|
| API fetch failure | `AlertCircle` icon + error message centered (matches agent health page pattern) |
| SSE disconnection | Silent reconnect; no user-facing error |

## State Management Contract

### New Store Slice: useDashboardStore (or extend useTaskStore)

```
interface DashboardState {
  // Filter state (synced with URL params)
  activeTab: "all" | "running" | "completed" | "failed";
  searchQuery: string;
  currentPage: number;

  // Task list from API
  tasks: Task[];
  totalTasks: number;
  totalPages: number;
  isLoading: boolean;

  // Per-task subtask counts (from API response or derived)
  taskProgress: Record<string, { completed: number; total: number }>;

  // SSE connection management
  multiTaskSSEDisconnect: (() => void) | null;

  // Actions
  loadDashboard: (params?: { status?: string; q?: string; page?: number }) => Promise<void>;
  setActiveTab: (tab: DashboardState["activeTab"]) => void;
  setSearchQuery: (q: string) => void;
  setPage: (page: number) => void;
  connectMultiTaskSSE: (taskIds: string[]) => void;
  disconnectMultiTaskSSE: () => void;
  handleMultiTaskEvent: (event: TaskEvent) => void;
}
```

The existing `useAgentStore` with `connectStatusSSE()` is reused as-is for agent dots.
