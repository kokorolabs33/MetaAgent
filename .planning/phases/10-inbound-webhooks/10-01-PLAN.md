<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External -> Ingest endpoint | Untrusted HTTP from GitHub/Slack/attackers hits /api/webhooks/inbound/{id} |
| Webhook payload -> LLM | Attacker-controlled text flows to orchestrator prompt |
| Webhook payload -> Database | Untrusted data written to tasks table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Spoofing | /api/webhooks/inbound/{id} | mitigate | HMAC-SHA256 signature verification via webhook.VerifyHMAC; reject unsigned or mis-signed requests with 401 |
| T-10-02 | Tampering | Webhook payload | mitigate | HMAC covers entire body; any modification invalidates signature |
| T-10-03 | Repudiation | Task creation via webhook | mitigate | webhook_deliveries table records every delivery with webhook_id and timestamp; tasks store created_by linking to webhook creator |
| T-10-04 | Information Disclosure | Webhook secrets in API responses | mitigate | Mask secret and previous_secret as "***" in List/Get responses; only show full secret once on Create |
| T-10-05 | Denial of Service | /api/webhooks/inbound/{id} | mitigate | http.MaxBytesReader limits body to 1MB; webhook_deliveries cleanup prevents table bloat; inactive webhooks return 404 early |
| T-10-06 | Elevation of Privilege | Prompt injection via payload | mitigate | SanitizeForLLM wraps all webhook content in [EXTERNAL WEBHOOK CONTENT START/END] delimiters per D-04; title capped at 200 chars, description at 5000 chars; control characters stripped |
| T-10-07 | Replay | Duplicate webhook deliveries | mitigate | Idempotency via webhook_deliveries table with delivery_id primary key; ON CONFLICT DO NOTHING prevents duplicates per D-13/D-14 |
| T-10-08 | Spoofing | Secret rotation window | accept | Dual-secret scheme per D-03 accepts both current and previous secret; window is indefinite until admin rotates again; acceptable for MVP |
</threat_model>
